SpringBoot

发表于2023-07-02|更新于2023-07-07
|字数总计:1.1k|阅读时长:3分钟

SpringBoot其设计目的是用来简化Spring应用的初始搭建以及开发过程

配置

SpringBoot默认配置文件application.properties，通过键值对配置对应属性

  • 修改服务器端口

    server.port=80

  • SpringBoot提供了3种配置文件的格式

    • properties(传统格式)
    • yml（主流格式）
    • yaml

  • 配置文件加载优先级：application.properties>application.yml>application.yaml

  • 语法规则：

    • 大小写敏感
    • 使用缩进表示层级关系，同层级左侧对齐，只允许使用空格
    • 属性值前面添加空格
    • # 表示注释
    • 数据前面要加空格与冒号隔开
    • 使用~表示null

  • 数据读取使用@Value，属性名引用方式：${一级属性名.二级属性名......}

  • 属性值中如果出现转移字符，需要使用双引号包裹

  • 也可以使用Environment对象封装全部信息，使用@Autowired自动装配，使用env.getProperty("")

  • 自定义对象封装指定数据

整合

  • 整合JUnit

    • 范例：@SpringBootTest(classes = SpringbootJunitApplication.class)
    • classes设置SpringBoot启动类
    • 如果测试类在SpringBoot启动类的包或子包中，可以省略启动类的设置，即省略classes的设定

  • 整合MyBatis时如果MySQL驱动强制要求设置时区，则修改url，添加serverTimezone设定

为方便调试可以开启MyBatisPlus日志

MyBatis-Plus

  • @TableName注解用于指定表名

  • @TableId中的value属性用于指定主键，type属性用于修改主键生成策略，默认采用雪花算法，type = IdType.AUTO代表自增

  • @TableField("user_name")指定属性对应的字段名

  • 物理删除：真实删除，将对应数据从数据库删除，之后查询不到此条被删除的数据。逻辑删除：假删除，将对应数据中代表是否被删除字段的状态修改为“被删除状态”，在数据库中仍能看到此条记录，可以进行数据恢复。采用TableLogic来进行 逻辑删除注解

  • 条件优先级使用lambda表达式进行构造实现

雪花算法

值为一个long型，长度为64bit。

  • 首先是一个符号位，1bit标识，id一般为正数，所以id最高位是0
  • 41bit时间戳（毫秒级），存储的是时间戳的差值，（当前时间戳 - 开始时间戳），结果约等于69.73年
  • 10bit作为机器id（5bit为数据中心，5个bit的机器ID，可以部署在1024个节点）
  • 12bit作为毫秒内的流水号（意味着每个节点在每毫秒可以产生4096个ID）
  • 雪花算法得出来的id整体上按时间自增排序，并且整个分布式系统内不会产生ID碰撞，效率较高

运维实用

  • 编程完毕后执行打包，生成jar包，在windows中通过java -jar xxx.jar执行

  • 带属性启动SpeingBoot，java -jar springboot.jar --server.port=80，携带多个属性启动，属性间用空格分隔

  • 多环境开发用—隔开

  • 使用spring.profiles.active启动指定环境，spring.config.activate.on-profile来设置环境

  • 也可以使用独立配置文件定义环境属性，取名为application-xxx.yml，xxx即为环境名

  • 当Maven与SpringBoot同时对多环境进行控制时，以Maven为主，SpringBoot使用@..@占位符读取Maven对应的配置属性值

  • 使用日志

  • 或者使用@Slf4j注释直接生成log对象，使用log打印日志
