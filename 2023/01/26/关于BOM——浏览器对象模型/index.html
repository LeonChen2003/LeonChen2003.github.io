关于BOM——浏览器对象模型

发表于2023-01-26|更新于2023-01-30
|字数总计:2k|阅读时长:6分钟

BOM概述

BOM即浏览器对象模型，提供了独立于内容而与浏览器窗口进行交互的对象，其核心对象是window

BOM由一系列的相关对象构成，并且每个对象都提供了许多属性与方法

BOM缺乏标准，JavaScript的标准化组织是ECMA，DOM的标准化组织是W3C，BOM最初是Netscape浏览器标准的一部分

DOM BOM
文档对象模型 浏览器对象模型
DOM就是把文档当作一个对象来看 把浏览器当成一个对象来看待
DOM的顶级对象是document BOM的顶级对象是window
DOM主要学习的是操作页面元素 BOM学习的是浏览器窗口交互的一些对象
DOM是W3C标准规范 BOM是浏览器厂商在各自浏览器上定义的，兼容性较差

BOM比DOM更大，它包含了DOM

window包含了document、location、navigation、screen、history

window对象是浏览器的顶级对象，它具有双重角色

  1. 它是js访问浏览器窗口的一个接口
  2. 它是一个全局对象。定义在全局作用域中的变量、函数都会变成window对象的属性和方法。在调用时省略window，前面所提到的对话框都属于window对象方法，如alert()、prompt()等等
  3. window下有一个属性window.name，因此name不应该用于变量的命名

window对象常见事件

窗口加载事件

window.onload是窗口页面加载事件，当文档内容全部加载完成时会触发该事件（包括图像、脚本文件、CSS文件等）

  • 有了window.onload就可以将js代码写到页面的元素的上方，onload会将页面内容全部加载完毕再去执行处理函数
  • window.onload传统注册事件只能写一次，如果有多个，会以最后一个为准
  • 如果使用addEventListener则没有限制

document.addEventListener("DOMContentloaded",function(){})事件触发时，仅当DOM加载完成时，不包括样式表、图片、flash等等。

如果页面图片很多的话，从用户访问到onload触发可能需要较长的时间，交互效果就不能实现，必然影响用户的体验，在此时用document.addEventListener("DOMContentloaded",function(){})更加的合适，不需要等待页面全部准备完成之后才能进行交互

调整窗口大小事件

winddow.onresize是调整窗口大小加载事件，当触发时就调用的处理函数。

定时器

setTimeout()定时器

window.setTimeout(调用函数，延迟的毫秒数)该方法用于设置一个定时器，在定时器到期后执行调用函数

  1. window可以省略
  2. 这个调用函数可以直接写函数，也可以写函数名
  3. 延迟的毫秒数默认为0，如果写，单位必须是毫秒

回调函数：

例如setTimeout()这种函数被称为回调函数callback,普通函数是按照代码顺序进行执行，而这个函数，需要等待时间，时间到了才去调用这个函数，因此称为回调函数

停止定时器：

采用clearTimeout(计时器的名称)实现对计时器的停止

setInterval()定时器

该函数使用方法与setTimeout()相同，但是不同的是该函数每隔一段时间就会反复的去调用给定的函数，而setTimeout执行一次之后便不再执行

停止定时器：

采用clearInterval(计时器的名称)实现对计时器的停止

JS执行机制

JS语言的一个特点就是单线程，同一个时间只能做一件事，比如对某个DOM元素进行添加和删除操作时，不能同时进行，单线程就意味着所有的任务需要排队，前一个任务结束才会执行下一个任务，这样将导致如果JS的执行时间过长，会造成页面的渲染不连贯，导致页面渲染加载阻塞

同步和异步：

为解决这个问题，利用多核CPU的计算能力，HTML5提出Web Worker标准，允许JS脚本创建多个线程，于是JS中出现了同步和异步。

同步：前一个任务执行完之后再执行下一个任务，程序的执行顺序与任务的排列顺序一致。

异步在执行一个任务的同时可以去处理其他的任务

  1. 先执行执行栈中的同步任务。
  2. 异步任务（回调函数）放入任务队列中。
  3. 一旦执行栈中所有同步任务完成，系统就会按次序读取任务队列中的异步任务，于是被读取的异步任务结束等待状态，进入执行栈，开始执行。

location对象

location属性用于获取和设置窗体的URL，并且可以用于解析URL，因为这个属性返回的是对象，有时也称location对象

URL：统一资源定位符，互联网上标准资源的地址，互联网上的每个文件都有一个唯一的URL，包含的信息指出文件的位置以及浏览器该如何执行它

URL的一般语法格式为protocal://host[:port]/path/[?query]#fragment

组成 说明
protocol 通信协议，如http、ftp、maito等等
host 主机（域名），如www.baidu.com
port 端口号，可选，省略时使用方案的默认端口，如http的默认端口为80
path 路径，由零或多个’/‘隔开的字符串，一般用来表示主机上的目录或者文件地址
query 参数，以键值对的形式，通过&符号隔开
fragment 片段，#后面内容，常见于连接、锚点

location对象属性：

对象属性 返回值
location.href 获取或者设置整个URL
location.host 返回主机（域名）
location.port 返回端口号，如果未写则返回空的字符串
location.pathname 返回路径
location.search 返回参数
location.hash 返回片段

location对象方法

对象方法 返回值
location.assign() 跟href一样，可以跳转页面（也称为重定向页面）（记录浏览历史）
location.replace() 替换当前页面，因为不记录历史，所以不能后退页面
location.reload() 重新加载页面，相当于刷新F5，如果参数为true则代表强制刷新

刷新与强制刷新：

F5 和 Ctrl+R 都是普通刷新，若页面之前访问过，就会发一个空请求到服务器，服务器返回 302 ，表示资源未更新，可以使用浏览器缓存。
强制刷新 (ctrl+F5) 则不管浏览器是否缓存，都要重新去源站服务器请求资源，成功则返回 200。

navigator对象

navigator对象包含有关浏览器的信息，它有很多属性，我们最常用的是userAgent，该属性返回由客户机发送服务器的user-agent头部的值，可以判断用户哪个终端打开页面，实现跳转

history对象

history对象与浏览器历史记录进行交互，该对象包含在浏览器窗口中访问过的URL

history对象 作用
back() 后退功能
forward() 前进功能
go(参数) 前进后退功能，参数为正代表前进，反之则代表后退
