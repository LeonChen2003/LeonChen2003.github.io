初识Node.js与内置模块

发表于2023-01-30|更新于2023-03-01
|字数总计:1.8k|阅读时长:6分钟

Node.js简介

Node.js是一个基于Chrome V8 引擎的 JavaScript 运行环境。

  • 浏览器是JavaScript的前端运行环境
  • Node.js 是JavaScript的后端运行环境
  • Node.js 中无法调用DOM和BOM等浏览器内置API

什么是终端(Terminal)：

是专门为开发人员设计的，用于实现人机交互的一种方式。

fs文件系统模块

fs模块是用来操作文件的模块。如果需要在js代码中使用fs模块来操作文件，则需使用const fs = require("fs")先进行导入

  • fs.readFile()方法，用来读取指定文件中的内容

    • fs.readFile(path[,options],callback)

    • 参数一为必选参数，字符串，表示文件的路径

    • 参数二可选，表示以什么编码格式来读取文件

    • 必选参数，文件读取完成后，通过回调函数拿到读取的结果

  • fs.writefile()方法，用来向指定的文件中写入内容

    • fs.writeFile(path,data[,options],callback)

    • 参数1为必选参数，需要一个指定文件路径的字符串，表示文件的存放路径

    • 参数2为必选参数，表示要写入的内容

    • 参数3为可选参数，表示以什么格式写入文件内容，默认值为utf-8

    • 参数4为必选参数，文件写入完成后的回调函数

整理成绩案例

路径动态拼接问题：

在使用fs模块操作文件时，如果提供的操作路径是以./或者../开头的相对路径时，很容易出现路径动态拼接错误的问题。（代码在执行时，会以执行node命令时所处的目录，动态拼接出被操作文件的完整路径）

因此，在使用fs模块操作文件时，直接提供完整的路径，不要提供./或者../开头的相对路径，从而防止路径动态拼接的问题

使用__dirname表示当前文件所处的目录

path路径模块

path模块是用来处理路径的模块。

  • path.join()方法，用来将多个路径片段拼接成一个完整的路径字符串

    • path.join([...paths])

    • ...paths<string> 路径片段的序列

    • 返回值：string

    • ```js
      const fs = require(‘fs’);
      const path = require(‘path’);

      fs.readFile(path.join(__dirname,’/1.txt’),’utf-8’,function(err,dataStr){
      console.log(dataStr);
      })

模块化

遵守固定规则，将大文件拆成独立并相互依赖的多个小模块

  • 提高复用性
  • 提高可维护性
  • 实现按需加载

模块分类

  • 内置模块
  • 自定义模块（用户自己的js文件）
  • 第三方模块

模块作用域

和函数作用域类似，在自定义模块中定义的变量、方法等成员，只能在当前模块内被访问，这种模块级别的访问限制，叫做模块作用域

在自定义模块中，使用module.exports对象将模块内的成员共享，供外界使用，使用require()方法导入的便是module.exports()所指定的对象

Node提供了exports对象，默认情况下，其与module.exports指向同一个对象，使用require()模块时，得到的都是module.exports()对象

  • 每个模块内部，module变量代表当前模块
  • module变量是一个对象，它的exports属性是对外的接口
  • 加载某个模块，其实就是加载module.exports属性

npm与包

在VSCode出现npm使用时找不到路径并要求你选择打开方式时参考这个链接 运行 npm 会弹出询问 “你要如何打开这个文件?“_火兰的博客-CSDN博客](https://blog.csdn.net/qq_47443027/article/details/120695450))

第三方模块和包的意义相同

node_modules 文件夹用来存放所有已安装到项目中的包。require()导入第三方包时，就是从这个目录中查找并加载这个包

package-lock.json配置文件用来记录node_modules目录下的每一个包的信息，例如包的名字，版本号，下载地址等

包的语义化版本规范

  • 第一位数字：大版本
  • 第二位数字：功能版本
  • 第三位数字：bug修复版本

前面的版本号增长，后面的版本号归零。
